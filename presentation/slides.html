2. Arquitectura del Pipeline

Presentado por: Francisco (Rol 4: Orchestration & Ops)

📂 Local Machine → Terraform (Infrastructure as Code)
▼
🪣 Amazon S3 — Data Lake con 3 zonas: raw/ processed/ curated/
▼
⚙️ AWS Glue Job — Transforma CSV → Parquet (particionado por year)
▼
🔍 AWS Glue Crawler → Glue Data Catalog (Schema-on-Read)
▼
📊 Amazon Athena (SQL) → Amazon QuickSight (Dashboard)

Orquestación

AWS Step Functions coordina: Glue Job → Crawler → Validación Athena (con manejo de errores)

Seguridad

Usuarios IAM con PowerUserAccess • Rol de servicio GlueCrawlerRole • Credenciales protegidas por .gitignore

3. Demo: Pipeline End-to-End

Presentado por: Diego (Rol 1: Data Engineer)

Paso 1: Ingesta con Terraform

$ cd pipeline/terraform
$ terraform init
$ terraform apply → Crea S3 + sube 4 CSV + configura Glue

Paso 2: Transformación (Glue Job)

  • Lee los 4 CSV de raw/ con PySpark
  • Elimina registros sin año (dropna)
  • Convierte a Parquet y particiona por year
  • Escribe en processed/

Paso 3: Catalogación (Crawler)

  • El Crawler detecta automáticamente los esquemas
  • Crea tablas en fisheries_db_diego
  • Las tablas quedan listas para Athena (Schema-on-Read)

Resolución de Esquemas

Los 4 archivos tienen entre 9 y 17 columnas. Se mantuvieron como tablas independientes y la integración se resuelve en la capa analítica (vistas SQL).

6. Lecciones Aprendidas y Limitaciones

✅ Lo que salió bien

  • Terraform nos permitió reproducir toda la infraestructura con un solo comando
  • El particionamiento por año redujo los costos de Athena hasta un 99.5%
  • Las reglas de calidad detectaron problemas reales (duplicados en los 4 datasets)
  • Step Functions automatizó el pipeline completo sin intervención manual

❌ Limitaciones

  • Los datos de Sea Around Us no se actualizan en tiempo real
  • QuickSight tiene limitaciones en la capa gratuita
  • No implementamos data lineage ni versionado de datos
  • El esquema varía entre fuentes (9 a 17 columnas)

🔮 Mejoras Futuras

  • Implementar AWS Lake Formation para gobernanza de datos centralizada
  • Agregar notificaciones SNS cuando el pipeline falle
  • Materializar vistas en la zona curated/ con Athena CTAS
  • Integrar Great Expectations para validaciones más robustas
